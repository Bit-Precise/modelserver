# GPT-5.6 Family Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Register OpenAI's GPT-5.6 family (`gpt-5.6-sol` / `gpt-5.6-terra` / `gpt-5.6-luna`) in the model catalog, seed subscription rates into every plan and rate-limit policy, and mirror the rates into the dashboard defaults and the admin utilization analyzer — so clients can send `model: "gpt-5.6"` immediately after deploy.

**Architecture:** Data-only change. Structural twin of migration `045_add_gpt_5_4_mini_nano.sql`, with two new wrinkles carried through consistently: (1) OpenAI 5.6 introduces a non-zero `cache_creation_rate` (`= input × 1.25`), and (2) Short/Long context is a 2×/1.5× multiplier over the base — both directly representable by the existing `CreditRate.CacheCreationRate` field and `LongContextCreditRate` block. Existing `ApplyLongContextCreditRate` (policy.go:195) already multiplies both cache classes by `InputMultiplier`, so no Go logic changes.

**Tech Stack:** PostgreSQL SQL migration (embedded via `internal/store/migrations/*.sql`), Go 1.x, React/TypeScript dashboard (Vite).

## Global Constraints

- **Long-context threshold:** `threshold_input_tokens = 272000` for all three models (project convention from `032`/`045`; OpenAI has not published the 5.6 cutoff).
- **Long-context multipliers:** `input_multiplier = 2.0`, `output_multiplier = 1.5` (matches OpenAI Long tier for all four token classes via `ApplyLongContextCreditRate`).
- **Catalog rate:** `credit = official-USD-price / 7.5` (project convention, `035_seed_catalog_default_credit_rates.sql`).
- **Plan / policy subscription rate:** `catalog × 0.2` (gpt-5.x family rate from `047` × 2 from `053`).
- **Aliases:** only `gpt-5.6-sol` takes alias `gpt-5.6`. `terra` / `luna` have no alias.
- **Publisher:** `openai`. **Status:** `active`. **Metadata:** `{"context_window":1050000,"category":"chat"}`.
- **Idempotency:** every DB `INSERT` uses `ON CONFLICT (name) DO NOTHING`; every `UPDATE plans/rate_limit_policies` uses a `NOT (model_credit_rates ? '<model>')` guard so re-runs and pre-existing operator overrides are preserved.

		long.CacheReadRate != 0.0268 || long.CacheCreationRate != 0.3336 {
		t.Fatalf("long: %+v; want 2x/1.5x", long)
